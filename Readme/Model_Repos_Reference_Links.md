## Notes

- **Transformers**: Can apply runtime quantization (4bit/8bit) if not pre-quantized
- **FP8 Models**: Pre-quantized, set "Quantized" to Yes in template
- **GGUF**: Must specify context_size, no runtime quantization available
- **Ollama**: Auto-pulls on first use, models stored in Docker volume
- **Vision**: QwenVL supports image + video, Florence-2 image only, LLM text-only
- **WD14**: ONNX classifiers, ~300-380MB each, CUDA or CPU, ~0.5-2s per image. Users can add any compatible SmilingWolf model by creating a template JSON with the HuggingFace repo ID.
